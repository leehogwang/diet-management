# 이미지 색감 보정 앱

Flutter로 개발된 이미지 색감 보정 앱입니다. 여러 가지 색감 보정을 적용하여 이미지를 증강하고, 최빈값 기반 합성을 통해 최종 결과 이미지를 생성합니다.

## 주요 기능

### 1. 이미지 선택
- 갤러리에서 이미지를 선택할 수 있습니다
- `image_picker` 패키지를 사용하여 구현

### 2. 색감 보정을 통한 이미지 증강
다음 조합으로 총 **27개**의 증강 이미지를 생성합니다:

- **밝기 조정**: -20%, 0%, +20% (3가지)
- **대비 조정**: 0.8, 1.0, 1.2 (3가지)
- **채도 조정**: 0.8, 1.0, 1.2 (3가지)
- **총 조합**: 3 × 3 × 3 = 27개

### 3. 최빈값(Mode) 기반 이미지 합성
- 모든 증강된 이미지의 각 픽셀 위치 (x, y)에서 RGB 값을 수집
- 각 채널(R, G, B)별로 최빈값을 계산
- 최빈값으로 최종 결과 이미지 생성
- 노이즈 제거 및 이미지 품질 향상 효과

### 4. 직관적인 UI
- **이미지 선택 버튼**: 갤러리에서 이미지 선택
- **색감 보정 처리 버튼**: 이미지 처리 시작
- **진행 상황 표시**: 실시간 진행률 표시 (LinearProgressIndicator)
- **원본/결과 비교**: 원본 이미지와 결과 이미지를 나란히 비교
- **증강 이미지 그리드**: 27개의 증강된 이미지를 3x9 그리드로 미리보기
- **결과 저장 버튼**: 처리된 이미지를 저장

### 5. 성능 최적화
- **비동기 처리**: async/await를 사용한 비블로킹 처리
- **진행률 추적**: 실시간 진행 상황 표시
- **효율적인 메모리 관리**: 픽셀 단위 처리

## 주요 함수 설명

### ImageProcessor 클래스

#### `adjustBrightness(img.Image, double)`
- 이미지의 밝기를 조정합니다
- percentage: -100 ~ 100 (-20%, 0%, +20%)

#### `adjustContrast(img.Image, double)`
- 이미지의 대비를 조정합니다
- factor: 0.5 ~ 2.0 (0.8, 1.0, 1.2)

#### `adjustSaturation(img.Image, double)`
- 이미지의 채도를 조정합니다
- factor: 0.0 ~ 2.0 (0.8, 1.0, 1.2)

#### `generateAugmentedImages(img.Image, callback)`
- 원본 이미지로부터 27개의 증강 이미지를 생성합니다
- 밝기, 대비, 채도의 모든 조합을 적용
- 진행률 콜백 제공

#### `synthesizeByMode(List<img.Image>, callback)`
- 최빈값 기반으로 이미지들을 합성합니다
- 각 픽셀 위치에서 RGB 채널별 최빈값 계산
- 진행률 콜백 제공

#### `_calculateMode(List<int>)`
- 정수 리스트에서 최빈값을 계산합니다
- 동일한 빈도인 경우 평균값 사용

## 사용 방법

1. **이미지 선택**: "이미지 선택" 버튼을 클릭하여 갤러리에서 이미지를 선택합니다
2. **처리 시작**: "색감 보정 처리" 버튼을 클릭하여 이미지 처리를 시작합니다
3. **진행 확인**: 진행률 바를 통해 처리 진행 상황을 확인합니다
4. **결과 확인**: 원본 이미지와 결과 이미지를 비교하고, 증강된 이미지들을 확인합니다
5. **저장**: "결과 이미지 저장" 버튼을 클릭하여 결과를 저장합니다

## 알고리즘 설명

### 최빈값 기반 이미지 합성
1. 원본 이미지에 27가지 색감 보정을 적용하여 27개의 변형 이미지 생성
2. 각 픽셀 위치 (x, y)에 대해:
   - 27개 이미지에서 해당 위치의 RGB 값을 수집
   - R, G, B 각 채널별로 최빈값(가장 많이 등장하는 값) 계산
   - 최빈값을 결과 이미지의 픽셀 값으로 설정
3. 이를 통해 노이즈가 제거되고 안정적인 이미지 생성

## 성능 고려사항

- **이미지 크기**: 큰 이미지(1920x1920)의 경우 처리 시간이 수 초~수십 초 소요될 수 있습니다
- **메모리**: 27개의 이미지를 동시에 메모리에 유지하므로, 매우 큰 이미지는 메모리 부족 발생 가능
- **최적화**: 필요시 isolate를 사용한 백그라운드 처리로 개선 가능
